# Opera UI 设计规范文档

## 0. 2026-04-03 三入口工作台补充规范

### 0.1 产品定位

- 全局产品名统一为「内容创作工具」
- 工作台分为三个独立入口：`内容转换`、`原创创作`、`微信公众号`
- `内容转换` 延续现有暖橙视觉与结果模块顺序，不改旧链路
- `原创创作` 使用暖橙主色 + 紫色辅助强调，突出“从零创作”的编辑器感
- `微信公众号` 使用偏微信生态的绿色强调，突出“长文写作 + 待发草稿箱”心智

### 0.2 三 Tab 信息架构

```text
Header（Opera / 内容创作工具）
└── TabNav
    ├── 内容转换：公众号长文 → 小红书内容包
    ├── 原创创作：主题想法 → 标题 / 正文 / 标签 / 配图建议
    └── 微信公众号：主题想法 → 标题 / 摘要 / 正文 / 本地待发草稿箱
```

- Tab 位于 Header 下方，桌面端居中，移动端横向紧凑排列
- 激活态使用底部高亮线和更高对比度文字
- `内容转换` 使用主色高亮，`原创创作` 使用 accent 高亮，`微信公众号` 使用 emerald 高亮
- 非激活态保持低饱和，避免与主操作按钮抢焦点

### 0.3 原创创作页布局

1. 顶部说明：强调“告诉 AI 你想写什么，帮你写出一篇小红书帖子”
2. 参数区：主题、内容类型、调性、目标字数、模型配置
3. 进度区：共 4 个可见阶段 `提炼主题 / 生成标题 / 撰写正文 / 标签配图`
4. 编辑区：标题、正文、配图建议、标签四块卡片逐步出现
5. 操作区：主按钮为「复制全文」，次按钮仅显示置灰「即将上线」

### 0.4 Composer 交互要求

- 标题先返回整块结果，正文按真实 SSE 增量持续增长，标签与配图建议最后返回
- 标题、正文、标签都要支持块级重生成入口
- 正文使用可扩展 textarea；标签支持增删与手动补充
- 配图建议保持文字提示形态，不做上传与图库能力
- 发布相关按钮不显示可用态，不出现“已接入发布”的误导文案

### 0.5 微信公众号页布局

1. 顶部 Hero：强调“从选题生成完整公众号文章”，使用浅绿色渐变 + 数据摘要卡片
2. 参数区：主题、文章类型、调性、篇幅、模型配置
3. 进度区：共 4 个可见阶段 `定位选题 / 生成标题 / 撰写摘要 / 生成正文`
4. 编辑区：标题、摘要、正文三块卡片逐步出现，支持块级复制与重生成
5. 草稿区：右侧独立面板展示当前草稿状态、保存入口、最近草稿和待同步说明

### 0.6 验收视觉基线

- Header 与 Tab 构成同一工作台壳层，整体文案统一为「内容创作工具」
- `内容转换` 页面保持当前模块节奏：标题、卡片、正文、标签
- `原创创作` 页面结果卡片需明显区分块级编辑与复制动作
- `微信公众号` 页面需体现更强的编辑器感和本地草稿箱语义，不得误导为真实公众号已接入
- 进度条组件允许注入步骤配置，但视觉风格保持统一

## 1. 页面整体布局设计

### 1.1 信息架构

页面采用单栏垂直流式布局，信息层级从上到下依次为：

```
┌─────────────────────────────────────┐
│  Header（品牌标识 + 产品说明）        │  固定顶部，高度 56px
├─────────────────────────────────────┤
│  页面标题 + 产品描述                  │  居中展示
├─────────────────────────────────────┤
│  输入区（大文本框 + 字数/段落统计）    │  核心交互入口
├─────────────────────────────────────┤
│  调性选择器（3 种预设卡片）           │  生成前必选
├─────────────────────────────────────┤
│  生成按钮 + 重新生成按钮              │  主操作区
├─────────────────────────────────────┤
│  进度指示器（5 步骤进度条）           │  生成中显示
├─────────────────────────────────────┤
│  输出区                              │  渐进渲染
│  ├─ 一键复制全部                     │
│  ├─ 封面标题（3-5 个备选）           │
│  ├─ 卡片文字（4-6 张）              │
│  ├─ 正文文案                        │
│  └─ 标签推荐（三层分组）             │
├─────────────────────────────────────┤
│  Footer                             │
└─────────────────────────────────────┘
```

### 1.2 桌面端布局（>= 640px）

- 内容区最大宽度：896px（max-w-4xl），水平居中
- 左右内边距：24px（sm:px-6）
- 调性选择器：三列等宽网格（grid-cols-3）
- 卡片文字：两列网格（grid-cols-2）
- 上下间距：40px（py-10）

### 1.3 移动端布局（< 640px）

- 内容区满宽，左右内边距：16px（px-4）
- 调性选择器：单列堆叠（grid-cols-1）
- 卡片文字：单列堆叠（grid-cols-1）
- 上下间距：24px（py-6）
- Header 中的副标题隐藏（hidden sm:inline）

## 3. 色彩系统和视觉规范

### 3.1 设计理念

Opera 的视觉风格定位为"专业温暖"——既有工具类产品的克制和效率感，又带有内容创作的温度和亲和力。

核心原则：
- 用暖色调传递创作的热情，用大量留白保持专业感
- 避免过度装饰，让内容本身成为视觉焦点
- 通过微妙的阴影和圆角营造柔和、现代的界面质感

### 3.2 主色（Primary）

选用暖橙色系，传达创作的活力和温暖感。

## 4. 关键交互流程

### 4.1 完整用户旅程

```
[空状态]
  │
  ├─ 用户粘贴文章 ──→ 显示字数统计 + 段落识别
  │                    输入框边框变为 primary-300
  │
  ├─ 用户选择调性 ──→ 卡片高亮，显示选中勾选
  │                    生成按钮从灰色变为渐变橙色
  │
  ├─ 点击"开始生成" ──→ 按钮变为"生成中..."（旋转图标）
  │                      输入区和调性选择器禁用
  │                      进度指示器出现
  │
  ├─ 阶段 1（1.2s）──→ "正在提取核心观点..."
  │                      进度条第一段亮起
  │
  ├─ 阶段 2（2.4s）──→ "正在生成封面标题..."
  │                      封面标题模块渐进出现（slide-up）
  │                      页面自动滚动到输出区
  │
  ├─ 阶段 3（4.2s）──→ "正在生成卡片文字..."
  │                      卡片文字模块渐进出现
  │
  ├─ 阶段 4（5.8s）──→ "正在生成正文文案..."
  │                      正文文案模块渐进出现
  │
  ├─ 阶段 5（7.0s）──→ "正在推荐标签..."
  │                      标签推荐模块渐进出现
  │                      进度指示器消失
  │                      "一键复制全部"按钮出现
  │                      "重新生成"按钮出现
  │
  └─ [完成状态]
       ├─ 用户可逐条复制/编辑
       ├─ 用户可一键复制全部
       └─ 用户可点击"重新生成"重置
```

### 4.2 状态转换矩阵

| 当前状态 | 触发条件 | 目标状态 | UI 变化 |
|----------|----------|----------|---------|
| 空状态 | 粘贴文本 | 就绪（部分） | 输入框高亮，显示统计 |
| 就绪（部分） | 选择调性 | 就绪（完全） | 生成按钮激活 |
| 就绪（完全） | 点击生成 | 生成中 | 按钮变加载态，输入禁用 |
| 生成中 | 各阶段完成 | 生成中 | 渐进渲染输出模块 |
| 生成中 | 全部完成 | 完成 | 进度消失，显示完成提示 |
| 完成 | 点击重新生成 | 就绪（完全） | 清除输出，恢复输入 |
| 完成 | 清空输入 | 空状态 | 全部重置 |

### 4.3 错误状态处理

| 场景 | 处理方式 |
|------|----------|
| 输入为空点击生成 | 按钮禁用态，不可点击 |
| 未选择调性点击生成 | 按钮禁用态，不可点击 |
| 复制失败 | 使用 execCommand fallback |
| 生成中断 | 重新生成按钮可重置状态 |

### 4.4 空状态设计

当没有生成结果时，显示引导性空状态：
- 64x64px 圆角图标（文档图标，neutral-300）
- 主文案："粘贴文章并选择调性后即可开始生成"（14px，neutral-400）
- 副文案："生成结果将按 封面标题 / 卡片文字 / 正文文案 / 标签推荐 依次展示"（12px，neutral-300）

## 5. 响应式适配方案

### 5.1 断点策略

采用 Tailwind CSS 移动优先（Mobile First）方案：

| 断点 | 宽度 | 适用设备 |
|------|------|----------|
| 默认 | < 640px | 手机竖屏 |
| sm | >= 640px | 手机横屏、小平板 |
| md | >= 768px | 平板 |
| lg | >= 1024px | 桌面端 |

### 5.2 关键适配差异

| 组件 | 移动端（< 640px） | 桌面端（>= 640px） |
|------|-------------------|---------------------|
| 内容区宽度 | 100%，px-4 | max-w-4xl，px-6 |
| 页面标题 | 24px（text-2xl） | 30px（text-3xl） |
| 调性选择器 | 单列堆叠 | 三列网格 |
| 卡片文字 | 单列堆叠 | 两列网格 |
| Header 副标题 | 隐藏 | 显示 |
| 上下间距 | py-6 | py-10 |
| 产品描述 | text-sm | text-base |

### 5.3 触摸适配

- 所有可点击元素最小触摸区域：44x44px
- 标签 Chip 有足够的内边距确保可点击性
- 按钮有 active:scale 反馈
- 输入框有足够的内边距方便触摸输入

## 6. 无障碍设计

### 6.1 WCAG AA 合规

- 所有文字与背景的对比度 >= 4.5:1
- 大文字（>= 18px bold 或 >= 24px）对比度 >= 3:1
- 所有交互元素有 focus-visible 样式（2px outline，primary-500）
- 所有按钮有 aria-label
- 调性选择器使用 aria-pressed 标记选中状态
- 封面标题使用 role="button" 和键盘事件支持

### 6.2 键盘导航

- Tab 键可遍历所有交互元素
- Enter/Space 可激活按钮和选择器
- 输入框有关联的 label 和 aria-describedby

### 6.3 减少动画

- 所有动画使用 CSS 实现，可通过 prefers-reduced-motion 媒体查询禁用
- 动画时长控制在 500ms 以内，避免眩晕感
