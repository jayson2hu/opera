🎭 Opera 产品体验报告

基于三类核心用户的真实使用场景模拟 · 2026-05-12

Executive Summary

Opera 的核心价值主张清晰：帮助公众号创作者把存量内容快速适配到小红书。产品已具备完整的三流程（公众号创作、改写适配、小红书原创），技术实现扎实。

当前版本存在三个系统性问题：

1. 新用户不知道从哪里开始 — Tab 关系和工作流顺序不清晰

2. 关键决策点缺乏上下文引导 — 语气、类型、排版选择时缺少参考

3. 生成结果的“下一步”路径不清晰 — 复制后不知道怎么用，流程间缺少行接

这些问题不需要重构，只需要在现有框架内做精准的信息补充和流程优化。

李晓雯 · 职场知识博主

公众号 1.2 万粉 · 每周更新 · 小红书几乎没有运营

场景：想把过去 3 个月的存量文章批量改写成小红书内容

使用路径：进入产品 → 看到三个 Tab → 选 公众号转小红书 → 粘贴文章 → 卡在语气选择
① 语气选择卡片（知识分享/轻松解释/朋友推荐）描述太抽象，不知道哪个适合自己的职场干货风格
② 生成 6 个封面标题后，不知道哪个最适合，没有任何筛选引导
③ 复制全部后，不清楚各部分（封面标题/卡片/正文/标签）分别对应小红书发布界面的哪个位置
④ 改写完一篇后，没有“继续改写下一篇”的快速路径，需要手动清空重来
陈

陈默 · 教育培训讲师

线下课程讲师 · 刚开始做内容 · 想同时运营公众号和小红书

场景：第一次使用，想生成一篇公众号文章，再把它改写成小红书

使用路径：进入产品 → 不确定从哪个 Tab 开始 → 选了 微信公众号 → 生成文章 → 想转小红书但找不到入口
① 三个 Tab 的关系和使用顺序不清晰，新用户不知道工作流是什么
② 公众号文章生成完后，没有“一键转小红书改写”的路径，需要手动复制再切换 Tab 粘贴

张

张薇 · 生活方式创作者

小红书原生用户 · 3000 粉 · 想用 AI 提升发帖效率

场景：想直接用小红书原创功能生成一篇关于“咖啡馆工作”的帖子

使用路径：进入产品 → 选 小红书原创 → 面对空白选题框不知道怎么写 → 填了简单几个字被提示“太短” → 不知道排版模板的区别
① 选题输入框的 placeholder 示例太专业（“独立开发者 AI 内容工作流”），与生活方式用户的内容距离感大
② “至少 10 字”的限制没有解释为什么，用户感觉被卡住而不是被帮助
③ 排版模板（清爽分段/清单笔记/故事种草/教程步骤）在生成前就要选，但没有视觉预览
④ 图片建议关键词只能看，不能直接跳转搜索或一键复制到图库搜索

改进建议（按优先级排序）

P0 立即修复 — 影响核心转化

1. 语气选择增加"适合场景"标签

问题：三个语气卡片的描述太抽象，用户无法快速判断哪个适合自己

方案：在每个语气卡片下方增加一行"适合场景"标签：

知识分享 → 适合：经验总结、方法论、行业观察

轻松解释 → 适合：概念科普、工具介绍、答疑解惑

朋友推荐 → 适合：产品种草、生活分享、情感共鸣

涉及组件：ToneSelector.tsx — 在 TONE_OPTIONS 中增加 tags 数组

实现成本：极低 ~30min

2. 选题输入框增加分类示例灵感

问题：placeholder 示例太单一且偏技术，生活方式用户无法参考

方案：在 TopicInput 下方增加可折叠的"选题灵感"区，按内容类型展示示例：

种草推荐 → "分享 5 个让居家办公效率翻倍的桌面好物"

知识干货 → "给新手讲清楚基金定投到底怎么操作"

故事经验 → "裸辞三个月后我学到的 3 件事"

教程指南 → "手把手教你用 Notion 搭建个人知识库"

涉及组件：TopicInput.tsx — 增加可折叠的示例区域

实现成本：低 ~45min

3. 公众号文章生成后增加"转小红书"快捷入口

问题：用户完成公众号文章后，想改写成小红书需要手动复制再切换 Tab 粘贴

方案：在 WeChatPage 的结果区增加"改写为小红书"按钮，点击后自动切换到 adapter Tab 并填入生成的正文

涉及组件：WeChatPage.tsx + App.tsx（需要 Tab 切换 + 内容传递的状态提升）

实现成本：中 ~2h

P1 近期优化 — 影响用户留存

4. Tab 导航增加工作流说明

问题：三个 Tab 的关系和使用顺序不清晰，新用户不知道从哪里开始

方案：在 TabNav 下方增加一行小字说明当前 Tab 的用途：

微信公众号 → "从选题生成完整公众号文章，可保存草稿"

公众号转小红书 → "粘贴已有文章，改写成小红书格式"

小红书原创 → "直接从选题生成小红书帖子"

涉及组件：TabNav.tsx — 增加 subtitle 字段

实现成本：极低 ~15min

5. 排版模板增加视觉预览

问题：四个排版模板选择时没有预览，用户不知道选了会怎样

方案：每个模板卡片增加 3-4 行的文字结构示意（用占位符展示结构差异）：

清爽分段 → 短段落 + 空行分隔

清单笔记 → ✅ 要点1 / ✅ 要点2 / ✅ 要点3

故事种草 → 开头引入 → 转折 → 推荐 → 总结

教程步骤 → Step 1 → Step 2 → Step 3

涉及组件：ComposerLayoutPanel.tsx — 在 TEMPLATES 中增加 preview 字段

实现成本：低 ~1h

6. 封面标题增加"推荐"标记和选择引导

问题：生成 6 个封面标题后用户不知道选哪个

方案：对第一个标题标记"AI 推荐"，并在标题列表顶部增加一行提示"建议选择数字开头或有悬念感的标题，点击率更高"

涉及组件：CoverTitles.tsx — 增加推荐 badge 和顶部提示文案

实现成本：低 ~30min

P2 长期改进 — 提升产品深度

7. 改写完成后增加"继续改写"快速路径

问题：批量改写用户需要手动清空输入区重新开始

方案：在改写结果底部增加"清空并改写新文章"按钮，一键重置输入区并滚动到顶部

实现成本：极低 ~15min

8. 草稿箱状态文案优化

问题："待同步"暗示会自动同步到某处，但 V1 并不会，造成用户困惑

方案：将"待同步"改为"已保存到本地"，移除"后续接入公众号账号后再同步"的未来承诺文案

涉及组件：DraftBoxPanel.tsx — 修改 STATUS_COPY 对象

实现成本：极低 ~5min

9. 图片关键词可点击搜索

问题：图片建议关键词只能看，不能直接用于搜索配图

方案：将图片建议关键词改为可点击的链接，点击后在新标签页打开小红书搜索或 Unsplash 搜索

涉及组件：ImageSuggestion.tsx — 将 span 改为 a 标签

实现成本：极低 ~15min

10. 生成结果增加"使用指南"提示

问题：用户复制全部后不知道各部分对应小红书发布界面的哪个位置

方案：在结果区顶部增加一个可折叠的"如何使用"提示：

封面标题 → 小红书发布时的标题栏

图文卡片 → 用 Canva/美图 制作成图片

发布正文 → 小红书发布时的正文区

话题标签 → 小红书发布时添加话题

实现成本：低 ~30min

核心洞察总结

Opera 不缺功能，缺的是"上下文"。

产品的三个流程已经完整可用，但用户在每个决策点都缺少足够的信息来做出选择。这不是功能缺失，而是信息设计的问题。

改进方向不是"加更多功能"，而是：

1. 在选择前给参考 — 语气标签、排版预览、选题示例

2. 在完成后给方向 — 使用指南、下一步按钮、跨 Tab 衔接

3. 在困惑时给解释 — Tab 说明、状态文案、字数限制原因

这些改动的共同特点：实现成本极低，但对用户体验的提升是系统性的。
